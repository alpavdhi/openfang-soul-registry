# 📚 Soul Registry Pipeline: Deployment Guide (v3.0.0)

## 🚀 Обзор Проекта
Soul Registry Pipeline — это критически важная система для управления жизненным циклом, конфигурацией и деплоем более 35 специализированных агентов (Soul Agents) в нашей многоагентной системе.

Основная задача пайплайна — обеспечить **атомарный (Transactional)** и **реплицируемый** процесс обновления всего агентского ядра. Мы гарантируем, что обновление либо пройдет полностью, либо откатится до исходного состояния, исключая "полурабочий хаос".

---

## 🛠️ Архитектура Системы
Система состоит из трех ключевых компонентов:

1.  **Manifests (Manifests Directory):** Директория, содержащая TOML-файлы (`*.toml`). Каждый файл описывает полный `SOUL` (модель, способности, контекст) одного агента.
2.  **Core Logic (`toml_apply_v3.py`):** Ядро пайплайна. Это Python-скрипт, который реализует транзакционную логику:
    *   Получает состояние всех запущенных агентов через API.
    *   Сравнивает текущее состояние с манифестами.
    *   Выполняет последовательные команды `KILL` $\to$ `SPAWN` для каждого агента, логируя каждую операцию.
    *   В случае любой ошибки (API 500, таймаут и т.д.) прерывает процесс, имитируя **ROLLBACK**.
3.  **CLI Interface (`cli.sh`):** Универсальный Bash-обёртка. Скрывает сложность Python-скрипта и предоставляет стандартизированные, интуитивно понятные команды для пользователя.

## ⚙️ Руководство Пользователя (Usage)

### 💾 Предварительные требования
1.  Python 3.x
2.  Установленные зависимости (requests, toml-parser и т.д.).
3.  Доступ к локальному API агентов (`http://127.0.0.1:4200/api/agents`).

### 🏃 Запуск Пайплайна
Все команды должны выполняться из корневой директории `soul-registry/`.

#### 1. Проверка (Dry Run) — **Обязательно!**
Прежде чем применять изменения в продакшене, всегда запускайте в режиме Dry-Run. Это покажет, какие действия будут предприняты, без фактического изменения системы.

```bash
./cli.sh --dry-run
```

#### 2. Целенаправленный Деплой
Если нужно обновить только одного агента (например, только `architect`):

```bash
./cli.sh --dry-run --agent architect
```

#### 3. Live Деплой (Продакшен)
Применяет изменения ко всем агентам. Используйте только после успешного Dry Run и подтверждения.

```bash
./cli.sh --live
```

## 📈 Зоны Роста (Future Scope)
Следующие шаги по улучшению стабильности и автоматизации:

1.  **CI/CD Integration:** Настройка GitHub Actions для автоматического запуска `cli.sh --dry-run` при каждом `git push`.
2.  **Мониторинг:** Внедрение системы логирования и оповещений (например, через Slack Webhooks) о статусе деплоя.
3.  **Откат:** Реализация явной команды `rollback` для быстрого возврата к предыдущей, стабильной версии манифестов.
